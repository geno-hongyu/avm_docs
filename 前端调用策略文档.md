# SeedX 20260712 前端调用策略文档

本文面向 `seedx_20260712_v3`。旧 0627 请求字段和响应兼容层已删除，前端必须按本文件和《前端接口文档》适配。

## 1. 基础规则

- BSC Testnet `chainId = 97`。
- 普通响应为 `{code,message,data}`；列表统一读取 `data.items / total / limit / offset`。
- 金额使用十进制字符串，链上整数单位只出现在明确标注为 `Wei` 的字段中。
- 写操作必须先使用登录钱包请求 tx-params，再根据 `requiredApprovals` 授权，最后发送返回的 `target/value/calldata`。
- 只有 `preflight.readyToSubmit = true` 才允许唤起钱包发送交易。
- 交易回执成功不等于后台已完成索引；需要继续轮询交易状态或项目 indexing-status。

## 2. 初始化配置

页面启动先请求 `GET /api/config/public`。前端不得硬编码以下内容：

- Platform、USDTTest、Multisig、Infinity Adapter/Hook 地址。
- `platformAdmin / creationFeeAdmin / platformPayout` 三种角色。
- 启用的 `scaleTiers` 及其节点数、节点价格、`nodeSaleDuration`、治理时间和毕业时间配置。

当前配置会返回 `standard-debug` 与 `standard-prod-sim` 两个启用档位。建项页面只展示接口返回且 `enabled = true` 的档位。

## 3. 创建项目

调用顺序：

1. `POST /api/projects/preview`
2. `POST /api/projects/create-tx-params`
3. 按 `requiredApprovals` 检查或提交 USDTTest 授权
4. 再次请求 tx-params，确认 `preflight.readyToSubmit = true`
5. 钱包发送 `createProject`
6. 轮询 `GET /api/transactions/:hash/status`
7. 等待 `indexed = true` 后刷新项目列表和详情

创建请求没有 `nodeSaleDuration`。该值由档位决定，只能从 `scaleTierSnapshot.nodeSaleDuration` 展示。

`tokenSymbol` 先转大写后校验：长度 1 到 10，首字符必须为 `A-Z`，其余只能为 `A-Z/0-9`。

如果配置 Alt Token：

- `altTokenAddress` 与 `altTokenPerNode` 必须同时有效。
- 后端按 Alt Token 实际 `decimals` 编码，不按 18 位假设。
- 节点认购可能同时需要 USDTTest 和 Alt Token 两笔授权。
- 任一余额不足时，`balanceChecks[].sufficient = false`，前端禁止继续授权或发送。

## 4. 统一写交易流程

所有 tx-params 响应都应按以下顺序处理：

1. 检查 `preflight.walletType`。要求 EOA 的操作遇到 `contract` 必须阻止。
2. 展示 `balanceChecks` 中的代币符号、decimals、当前余额和差额。
3. 对 `requiredApprovals[].approvalRequired = true` 的项发送 ERC-20 `approve`。
4. 授权确认后重新请求 tx-params，不复用旧 calldata 或旧模拟结果。
5. 仅当 `approvalsComplete = true`、`simulationStatus = passed` 且 `readyToSubmit = true` 时发送交易。
6. `estimatedGas` 用于钱包 gas 提示，不作为成功保证。

后端使用当前登录钱包作为 `eth_call.from`，因此前端不要用另一个钱包发送已经生成的 calldata。

## 5. 项目状态与按钮

以接口的实时 `projectState` 为准：

| 状态 | 前端主要操作 |
|---|---|
| `node_sale` | 节点认购、到期失败标记 |
| `launch_failed` | 领取退款 |
| `trading` | 买卖、质押、节点锁定、治理、触发毕业 |
| `graduating` | 展示毕业公告和倒计时，条件满足后执行毕业 |
| `graduated` | 治理、领取/分配 Infinity LP 手续费、查看外部池仓位 |

按钮可用性必须随详情或 tx-params 模拟结果刷新，不能只根据页面进入时的状态缓存。

## 6. 节点认购

- 认购上限来自项目 `maxNodePurchaseBps` 和当前地址已认购数量。
- 节点售价、`nodeTargetStable`、`nodeTokenUnit`、`nodeSaleDeadline` 均以项目详情或链上状态为准。
- `nodeFundraiseStable` 表示已售节点对应的实际募集额，不是目标募集额。
- Alt Token 项目必须同时展示 Alt Token 每节点数量、decimals、余额和 allowance。

## 7. 交易与仓位

- 买入、卖出、质押买入先调用报价接口，再请求 action tx-params。
- 报价变化后必须重新模拟，不能按 `amount * currentPrice` 自行拼交易金额。
- 毕业后链上质押和债务已归零；仓位页以接口当前值为准，历史记录单独展示。
- Alt Token 的锁定、已领取、可领取数量已经按真实 decimals 返回。

## 8. 治理

治理列表 `GET /api/projects/:address/governance` 使用标准分页，只读取 `items`，不再读取旧 `proposals` 字段。

状态含义：

- `active`：仍在投票。
- `pending_tally`：投票已结束，等待计票。
- `passed`：已通过，等待 timelock 或执行。
- `rejected`：计票后拒绝，已结案。
- `executed`：已执行，已结案。

能力字段：

- `allowedParamIds`：当前项目状态下合约允许的参数。
- `recommendedParamIds`：UI 推荐展示的参数。
- `unavailableParamIds`：同类别已有未结案提案，暂不可再次发起。

Trading 允许 1 到 9；Graduated 合约允许 1 到 11，但 UI 推荐 10、11。只有完整节点可发起提案。同项目同 `paramId` 在旧提案进入 `rejected` 或 `executed` 前不能再发起。

投票是否通过必须看计票结果，不得仅比较同意票和反对票；合约还会校验快照投票权和通过条件。

## 9. 毕业与 Infinity

毕业流程：

1. `trigger_graduation` 生成 `triggerGraduation()`。
2. 展示公告的 `announceTime / executableAt / rollingVolume / turnoverRateBps`。
3. 公告期结束后请求 `execute_graduation`。
4. 后端只生成无参 `executeGraduation()`。
5. 索引完成后刷新 `/pool`，读取真实 `graduatedAt` 和 `infinityPosition`。

前端不得提交 `sqrtPriceX96/tickLower/tickUpper`。这些字段只会作为 `InfinityGraduated` 的结果出现在 `infinityPosition` 中。

`graduationWindowVolume` 是当前毕业窗口成交量；`graduationVolumeWindow` 是窗口秒数。不要再使用 `volume30d` 作为 API 字段名。

## 10. 后台管理

- `GET /api/admin/platform`：读取链上平台角色、两套档位和 Infinity 配置。
- `POST /api/admin/platform/scale-tier-tx-params`：仅 `platformAdmin` 钱包可生成并模拟 `setScaleTier`。
- `POST /api/admin/platform/creation-fee-tx-params`：生成 Multisig `submit` 包装交易。
- `GET /api/admin/multisig`：交易列表来自数据库分页，链上只核验 signer/threshold。
- 管理端所有列表继续按 `items/total/limit/offset` 读取。

## 11. 错误处理

- 401：登录会话失效，重新签名登录。
- 403：后台 RBAC 权限不足。
- `blocked_by_approval`：先授权并等待授权交易确认。
- `insufficient_balance`：显示 `shortfallWei` 对应的可读差额，不允许继续。
- `contract_wallet_not_allowed`：当前写操作受合约 `onlyEOA` 限制。
- 模拟失败：直接展示后端解析后的自定义错误，不发送交易。
